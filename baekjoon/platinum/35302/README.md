# 라면 숨기기

## 링크

[문제 바로가기](https://www.acmicpc.net/problem/35302)

## 사용 알고리즘

- 그리디
- 정렬

## 풀이

각 선택의 효율이 좋은 순서대로 정렬한 뒤, 지금까지 확보한 라면 수를 기준으로 이번 선택이 만드는 이득을 누적했다. 코드에서는 `(A, C)`를 정렬해 순서를 고정하고, 각 항목이 기여하는 값을 바로 더하는 방식으로 계산한다.

## 복잡도

- 시간: `O(K log K)`
- 공간: `O(K)`

## 한 줄 회고

조건을 직접 시뮬레이션하기보다, 어떤 순서가 유리한지 먼저 정리하는 편이 낫다.
