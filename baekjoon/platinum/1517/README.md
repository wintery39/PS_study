# 버블 소트

## 링크

[문제 바로가기](https://www.acmicpc.net/problem/1517)

## 사용 알고리즘

- 세그먼트 트리
- 정렬

## 풀이

값 기준으로 정렬한 뒤, 각 원소의 원래 위치를 보면서 앞쪽에 이미 놓인 원소 수를 세었다. 현재 위치보다 왼쪽에 아직 남아 있는 원소 개수를 누적하면 버블 소트의 swap 횟수를 구할 수 있다.

## 복잡도

- 시간: `O(N log N)`
- 공간: `O(N)`

## 한 줄 회고

swap을 직접 하지 않고 i < j 일 떄, A[i] > A[j] 인 값을 찾는 것이 중요한 점이었다.
