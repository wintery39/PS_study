# 가장 가까운 두 점

## 링크

[문제 바로가기](https://www.acmicpc.net/problem/2261)

## 사용 알고리즘

- 기하
- 정렬
- 공간 분할

## 풀이

점들을 정렬한 뒤, 적당한 크기의 격자 칸에 넣어 가까울 수 있는 칸들만 비교했다. 현재 최솟값보다 멀어질 수밖에 없는 후보는 바로 제외해서 전체 비교 수를 줄였다.
적당한 크기의 격자라는 뜻은 N개의 점이 20001 \* 20001 공간에 있을 때 가장 가까운 두 점의 최댓값이다. (최악의 경우의 수 가정)

## 복잡도

- 시간: `O(min(격자의 크기, N) * N)`
- 공간: `O(N)`

## 한 줄 회고

가까운 점 문제는 전부 비교하지 않고 후보를 얼마나 잘 줄이느냐가 핵심이다. 다른 풀이로는 분할 정복을 하는 풀이가 있으며 해당 풀이의 경우 병합 할 때 분할된 두 값중 하나로 되는 것이 아니라 분할된 값들과 병합된 상태에서의 최선의 값을 비교해서 병합한다는 특징이 있었다.
