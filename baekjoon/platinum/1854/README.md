# K번째 최단경로 찾기

## 링크

[문제 바로가기](https://www.acmicpc.net/problem/1854)

## 사용 알고리즘

- 다익스트라

## 풀이

각 정점마다 최단거리 하나만 저장하지 않고, 도착한 거리들을 `k`개까지 기록했다. 힙에서 작은 비용부터 꺼내며 각 정점에 대해 `k`번째로 방문되는 거리까지 허용하면, 모든 정점의 `k`번째 최단거리를 구할 수 있다.

## 복잡도

- 시간: `O(kM log (kM))`
- 공간: `O(kN)`

## 한 줄 회고

최단거리 하나를 저장하던 다익스트라를 대신 k개로 확장하였다.
