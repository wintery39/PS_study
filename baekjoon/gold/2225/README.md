# 합분해 

## 링크
[문제 바로가기](https://www.acmicpc.net/problem/2225)

## 사용 알고리즘
- DP

## 풀이
`prev[s]`를 현재까지 사용한 수의 개수(반복문을 돈 횟수)로 합 `s`를 만드는 경우의 수라고 두고, 
다음 수를 하나 더 사용할 때 `next[t] += prev[s]` (`s <= t`) 형태로 누적한다. 
초기에는 수 1개로 각 합을 만드는 방법이 1가지이므로 `prev`를 1로 채운다.
오버플로우를 막기 위해 매 단계마다 `1_000_000_000`으로 나머지 연산을 적용한다. 

## 복잡도
- 시간: O(k*n^2) 
- 공간: O(n)
